# Release Checklist - 发布检查清单

## 概述

固件正式发布前的系统性检查，确保质量和完整性。

## 发布前检查

### 1. 代码检查

- [ ] 所有 TODO/FIXME 已处理
- [ ] 调试代码已移除
- [ ] 版本号已更新
- [ ] 代码审查已完成

### 2. 测试检查

- [ ] 单元测试全部通过
- [ ] 集成测试全部通过
- [ ] 回归测试无新问题
- [ ] 性能测试达标

### 3. 文档检查

- [ ] 版本更新日志
- [ ] API 文档更新
- [ ] 用户手册更新
- [ ] 发布说明准备

### 4. 构建检查

- [ ] Release 配置编译
- [ ] 优化选项正确
- [ ] 二进制大小合理
- [ ] 无编译警告

### 5. 安全检查

- [ ] 敏感信息已移除
- [ ] 调试接口已关闭
- [ ] 加密功能正常
- [ ] 签名验证通过

## 发布产物

| 产物 | 说明 |
|------|------|
| firmware.bin | 烧录文件 |
| firmware.hex | 备选烧录文件 |
| RELEASE_NOTES.md | 发布说明 |
| CHANGELOG.md | 变更日志 |
| checksum.md5 | 校验文件 |

## 版本号规范

```
v<主版本>.<次版本>.<补丁>[-<预发布标签>]

示例:
v1.0.0        - 正式版本
v1.0.1        - 补丁版本
v1.1.0        - 次版本更新
v2.0.0-beta   - 预发布版本
```

## 发布流程

```
1. 完成所有检查
      ↓
2. 创建发布分支/Tag
      ↓
3. 构建 Release 版本
      ↓
4. 生成发布产物
      ↓
5. 上传到发布服务器
      ↓
6. 通知相关人员
```

## 与 Core Workflow 集成

**在 Act 阶段调用:**
- 作为 `finishing-branch` 后的最终步骤
- 确保发布质量
