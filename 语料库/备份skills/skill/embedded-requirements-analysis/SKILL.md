# Requirements Analysis - 需求分析

## 概述

系统性地分析嵌入式项目需求，确保理解完整且无歧义。

## 需求分类

### 1. 功能需求

- 系统需要做什么？
- 输入是什么？
- 输出是什么？
- 有哪些操作模式？

### 2. 性能需求

| 维度 | 需要明确的内容 |
|------|---------------|
| **实时性** | 响应时间要求、周期性任务频率 |
| **吞吐量** | 数据处理速率、通信带宽 |
| **精度** | ADC 精度、控制精度 |

### 3. 资源约束

| 维度 | 需要明确的内容 |
|------|---------------|
| **成本** | BOM 成本目标、批量价格 |
| **功耗** | 电池寿命、待机功耗 |
| **尺寸** | PCB 面积、外壳限制 |
| **内存** | Flash/RAM 预算 |

### 4. 环境需求

- 工作温度范围
- 湿度、防护等级
- EMC 要求
- 认证要求

### 5. 接口需求

| 类型 | 需要明确的内容 |
|------|---------------|
| **用户接口** | 按键、显示、LED |
| **通信接口** | UART、SPI、I2C、CAN、无线 |
| **电源接口** | 供电方式、电压范围 |
| **机械接口** | 连接器类型、安装方式 |

## 输出格式

```yaml
项目名称: xxx
分析日期: YYYY-MM-DD

功能需求:
  核心功能:
    - [功能1描述]
    - [功能2描述]
  操作模式:
    - [模式1]: [描述]
    - [模式2]: [描述]

性能需求:
  实时性:
    最大响应时间: 10ms
    主循环周期: 1ms
  吞吐量:
    数据采集速率: 1kHz
    通信速率: 115200bps

资源约束:
  成本: ≤ 50 RMB
  功耗: 待机 < 1mA, 运行 < 100mA
  内存: Flash < 256KB, RAM < 64KB

接口需求:
  通信: UART x2, SPI x1
  用户: 按键 x3, LED x4
  电源: 3.3V / 5V

优先级排序:
  P0 (必须): [关键功能列表]
  P1 (重要): [重要功能列表]
  P2 (可选): [可选功能列表]

风险识别:
  - [潜在风险1及应对]
  - [潜在风险2及应对]
```

## 与 Core Workflow 集成

**在 Think 阶段调用:**
- 作为 `core-brainstorming` 的领域扩展
- 在理解项目目标后，进行详细需求分析

**输出流向:**
- 需求清单用于 `feasibility-assessment`
- 优先级用于 `writing-plans` 任务排序
