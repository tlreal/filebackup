# Platform Selection - 平台选型

## 概述

基于项目需求推荐合适的嵌入式硬件平台，对比性能、成本、生态、供货等维度。

## 选型流程

```
需求规格
    ↓
功能筛选 → 排除不满足功能的平台
    ↓
性能筛选 → 排除性能不足的平台
    ↓
成本筛选 → 排除超预算的平台
    ↓
生态评估 → 评估开发支持
    ↓
供货评估 → 评估供应链风险
    ↓
最终推荐
```

## 评估维度

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| 功能满足度 | 高 | 是否满足所有必需功能 |
| 性能余量 | 中 | CPU/内存是否有裕量 |
| 开发生态 | 中 | IDE、HAL库、示例代码 |
| 成本 | 中 | 单价、批量价格 |
| 供货稳定性 | 高 | 交期、替代料 |
| 功耗 | 可选 | 电池供电产品重点考虑 |

## 常见平台参考

### 低端 (成本敏感)
- STM32F0/F1 系列
- GD32 系列
- 国产 MCU

### 中端 (性能平衡)
- STM32F4/G4 系列
- ESP32 系列
- NXP LPC 系列

### 高端 (高性能)
- STM32H7 系列
- i.MX RT 系列
- ESP32-S3 (带 AI)

## 输出格式

```yaml
推荐方案:
  首选: STM32F407VGT6
  备选: STM32F405RGT6
  
对比分析:
  | 维度 | STM32F407 | STM32F405 | 需求 |
  |------|-----------|-----------|------|
  | 内核 | M4F 168MHz | M4F 168MHz | ≥100MHz ✓ |
  | Flash | 1MB | 1MB | ≥256KB ✓ |
  | SRAM | 192KB | 192KB | ≥64KB ✓ |
  | UART | 6 | 6 | ≥2 ✓ |
  | 单价 | ¥25 | ¥22 | ≤¥30 ✓ |

选型理由:
  1. 完全满足功能需求
  2. 性能有30%以上裕量
  3. 团队已有F4开发经验
  4. 供货稳定，有国产替代

风险提示:
  - 需确认USB Host功能是否需要
  - 大批量时可考虑GD32F407替代
```
