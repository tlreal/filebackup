# Prompt Engineering - 提示词工程

## 概述

有效的提示词是 AI 辅助开发的关键。三层提示词体系确保交互质量。

**核心原则:** 层次分明 + 专业准确 + 持续优化

## 何时使用

- 开始新的 AI 辅助开发任务
- 优化 AI 交互效果
- 建立项目提示词库
- 培训团队 AI 使用技能

## 三层提示词体系

```
系统层提示词
    ↓ 建立专业背景
阶段层提示词
    ↓ 匹配开发阶段
任务层提示词
    ↓ 完成具体任务
AI 输出
```

### 1. 系统层提示词

**作用:** 建立专业对话环境，确保输出质量

**核心要素:**
- 专业领域定义
- 技术背景说明
- 质量标准要求
- 基础规范约束

**示例:**
```
你是通信仿真系统专家，具有信号处理、通信协议和系统仿真等专业背景。
请基于以下技术标准和规范开展工作：
- 3GPP 标准规范
- IEEE 信号处理标准
- 公司内部开发规范
```

### 2. 阶段层提示词

**作用:** 匹配当前开发阶段，明确角色职责

| 阶段 | AI 角色 | 核心任务 |
|------|--------|---------|
| 需求分析 | 需求分析专家 | 需求获取与分析 |
| 架构设计 | 架构设计专家 | 系统架构设计 |
| 详细设计 | 技术专家 | 实现方案设计 |
| 代码开发 | 编程专家 | 代码实现 |
| 系统测试 | 测试专家 | 系统测试 |

**示例:**
```
现在处于架构设计阶段，需要你作为架构设计专家完成：
1. 分析系统架构风格选择
2. 进行系统模块分解
3. 设计关键接口
4. 评估质量属性
```

### 3. 任务层提示词

**作用:** 完成具体任务，获取直接可用的结果

**编写要点:**
- 任务目标明确
- 要求详细具体
- 参数清晰完整
- 易于验证结果

**示例:**
```
请针对 FSK 调制模块完成以下任务：
1. 技术需求：
   - 支持 2FSK/4FSK 调制
   - 载波频率：100kHz-10MHz
   - 符号速率：100bps-1Mbps
2. 输出要求：
   - 模块接口定义（Markdown格式）
   - 关键算法伪代码
3. 约束条件：
   - 基于 MATLAB 实现
   - 支持实时处理
```

## 提示词编写规范

### 基本要求

| 要求 | 说明 |
|------|------|
| 专业性 | 使用专业术语，符合行业标准 |
| 结构性 | 层次清晰，逻辑完整 |
| 可操作性 | 目标明确，结果可验证 |

### 编写原则

1. **专业导向原则**
   - 基于领域知识
   - 符合开发规范
   - 满足技术标准

2. **场景适配原则**
   - 匹配开发阶段
   - 适应任务需求
   - 考虑实际场景

3. **迭代优化原则**
   - 支持持续改进
   - 允许灵活调整
   - 方便版本管理

## 提示词优化方法

### 优化时机识别

**技术问题:**
- 专业性不够准确
- 方案可行性差
- 性能指标不达标

**效率问题:**
- 交互轮次过多
- 理解偏差明显
- 输出质量不稳定

### 优化方向

**技术准确性优化:**
```
优化前：
"实现一个FSK调制解调模块"

优化后：
"实现2FSK数字调制解调模块，要求：
1. 调制参数：
   - 载波频率：1kHz/2kHz
   - 码元速率：100bps-1kbps可配置
   - 采样率：10kHz
2. 性能指标：
   - BER<1e-5@SNR=10dB
   - 时延<100ms
3. 接口要求：
   - 输入：比特流
   - 输出：调制信号/解调比特流"
```

**完整性优化:**
```
优化前：
"分析系统性能"

优化后：
"分析FSK通信系统性能，需包含：
1. 理论分析
   - 调制效率
   - 带宽利用率
   - 功率谱特性
2. 仿真验证
   - AWGN信道下的BER曲线
   - 多径信道下的性能劣化
3. 输出要求
   - 性能分析报告（Markdown格式）
   - 关键指标对比表格"
```

## 提示词模板库

### 需求分析模板
```
请对 [具体需求] 进行细化分析，包括：
1. 功能维度：核心功能、可选功能、约束条件
2. 性能维度：关键指标、测试方法、验收标准
3. 接口维度：输入输出、数据格式、时序要求
4. 质量维度：可靠性、安全性、兼容性要求
```

### 代码生成模板
```
请实现 [功能名称]：
技术规格：[具体参数]
接口要求：[输入输出]
性能要求：[指标]
代码规范：[注释、验证、边界处理]
```

### 评审模板
```
请对 [文档/代码] 进行评审，重点关注：
1. 完整性：功能完整、场景覆盖、指标明确
2. 准确性：术语规范、描述清晰、数值合理
3. 一致性：内部一致、接口匹配、逻辑连贯
4. 可测性：指标可测、方法可行、标准明确
```

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| AI 理解偏差 | 提供更多上下文和具体示例 |
| 输出不符合预期 | 明确输出格式和验收标准 |
| 专业性不足 | 加载系统层提示词，提供专业背景 |
| 效率低下 | 建立模板库，复用成功经验 |
