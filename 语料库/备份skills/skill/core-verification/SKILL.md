# Verification Before Completion - 完成前验证

## 概述

在声明任何工作完成之前，系统性地验证所有方面都已正确完成。

**核心原则:** 「我认为完成了」 ≠ 完成。必须有证据。

## 验证清单

### 1. 功能验证

- [ ] 所有需求都已实现
- [ ] 所有测试都通过
- [ ] 手动验证关键路径
- [ ] 边界情况已处理

### 2. 代码验证

- [ ] 代码已审查
- [ ] 没有 TODO 或 FIXME 未处理
- [ ] 没有调试代码遗留
- [ ] 代码风格一致

### 3. 文档验证

- [ ] 必要的文档已更新
- [ ] API 文档准确
- [ ] README 更新（如需要）

### 4. 集成验证

- [ ] 与现有代码集成正常
- [ ] 没有破坏现有功能
- [ ] 性能在可接受范围

## 嵌入式领域扩展

**硬件相关验证:**
- [ ] 在目标硬件上测试通过
- [ ] 时序要求满足
- [ ] 功耗在预算内
- [ ] 内存使用在限制内

**系统级验证:**
- [ ] 中断处理正确
- [ ] 资源释放完整
- [ ] 看门狗正常

## 红线

**绝不在以下情况下声明完成:**
- 「应该可以工作」但没验证
- 测试跳过了某些情况
- 「以后再修」的问题存在
- 只在开发环境测试过

## 输出格式

```markdown
## 验证报告

### 功能验证
- ✅ 需求 1: xxx - 验证通过
- ✅ 需求 2: xxx - 验证通过
- ✅ 边界情况: xxx - 验证通过

### 代码验证  
- ✅ 代码审查: 已由 xxx 审查
- ✅ 无遗留 TODO
- ✅ 风格一致

### 测试结果
- ✅ 单元测试: 12/12 通过
- ✅ 集成测试: 5/5 通过

### 结论
所有验证项通过，可以标记为完成。
```
