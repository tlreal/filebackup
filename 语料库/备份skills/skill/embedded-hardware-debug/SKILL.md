# Hardware Debug - 硬件调试

## 概述

系统性地调试硬件相关问题，包括信号完整性、时序和外设配置。

## 调试工具

| 工具 | 用途 |
|------|------|
| **示波器** | 信号时序、波形分析 |
| **逻辑分析仪** | 数字信号、协议分析 |
| **万用表** | 电压、电阻测量 |
| **仿真器/调试器** | 代码调试、寄存器查看 |
| **串口工具** | 日志输出、交互调试 |

## 调试流程

### 1. 问题定义

- 清晰描述问题现象
- 确定问题是否可复现
- 收集相关信息（代码版本、硬件版本）

### 2. 隔离问题

```
问题在软件层？ → 检查代码逻辑
       ↓
问题在驱动层？ → 检查寄存器配置
       ↓
问题在硬件层？ → 检查电路和信号
```

### 3. 常见问题检查清单

**电源相关:**
- [ ] 电源电压正确
- [ ] 去耦电容足够
- [ ] 电源噪声在允许范围

**时钟相关:**
- [ ] 时钟源正确配置
- [ ] PLL 锁定
- [ ] 外设时钟使能

**GPIO 相关:**
- [ ] 引脚模式正确
- [ ] 复用功能正确
- [ ] 上下拉配置正确

**通信相关:**
- [ ] 波特率/时钟匹配
- [ ] 电平标准兼容
- [ ] 协议配置正确

### 4. 信号测量

**SPI 调试:**
```
测量点：
- CS 信号：低电平有效，建立时间
- CLK 信号：频率、占空比、相位
- MOSI/MISO：数据建立/保持时间
```

**I2C 调试:**
```
测量点：
- SDA/SCL 上拉电压
- 时钟频率
- ACK/NACK 响应
```

**UART 调试:**
```
测量点：
- TX/RX 电平
- 波特率误差
- 帧格式正确性
```

### 5. 日志输出

```c
// 调试日志级别
#define LOG_ERROR   0
#define LOG_WARN    1
#define LOG_INFO    2
#define LOG_DEBUG   3

#define LOG_LEVEL   LOG_DEBUG

#define LOG(level, fmt, ...) \
    do { \
        if (level <= LOG_LEVEL) \
            printf("[%s] " fmt "\n", log_level_str[level], ##__VA_ARGS__); \
    } while(0)
```
