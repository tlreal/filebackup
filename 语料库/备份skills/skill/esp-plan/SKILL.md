# ESP-Plan - 极简法则·规划技能

## 概述

在头脑风暴后，**通过互动式方法**与用户一起确定产品的简化路径（价格简化、命题简化或组合策略），并设计相应的产品架构。包含选择层和设计层两个阶段。

**核心原则：** 
- 通过互动式评估确定简化路径
- 支持"价格简化"、"命题简化"、"组合策略"三种模式
- 设计与执行强关联
- 输出结构化规划文档

## 何时使用

```
完成头脑风暴？
    ↓
运行选择层测试
    ↓
明确简化路径
    ↓
启动设计层
    ↓
输出产品蓝图
```

**使用场景：**
- 完成 brainstorming 后需要确定产品方向
- 需要在价格简化和命题简化之间做出选择
- 需要将战略转化为可落地的设计方案
- 需要确保设计与执行强关联

**何时不使用：**
- 简化路径已经明确
- 仅需小幅优化而非系统性设计
- 紧急场景下需要快速决策

## 第一阶段：选择层

### 目标

判定你是"天生的价格简化者"还是"天生的命题简化者"

### 步骤 1：倾向测试

**关键问题：** 企业资源更擅长"砍成本，做规模"还是"做体验，做精致"？

**评估维度：**

| 维度 | 价格简化倾向 | 命题简化倾向 |
|------|-------------|-------------|
| **团队专长** | 供应链管理、成本控制、规模化生产 | 用户体验设计、快速迭代、品牌建设 |
| **成本结构** | 固定成本高，边际成本低 | 研发投入高，溢价能力强 |
| **品牌基因** | 大众市场、性价比、可靠性 | 高端市场、创新性、情感连接 |
| **核心能力** | 效率驱动、标准化、流程优化 | 创新驱动、差异化、个性化 |

**与用户互动：**
```
AI: 根据以下维度，请您评估您的团队/企业在哪个方面更强？
    - 团队专长：供应链管理 vs 用户体验设计
    - 成本结构：规模化生产 vs 研发投入
    - 品牌基因：性价比 vs 创新性
    
    请选择：价格简化倾向 / 命题简化倾向 / 两者都强 / 两者都弱
    
用户：[选择答案]
    
AI: 感谢您的回答。倾向测试结果：[结论]
```

**输出：** 倾向测试报告（明确倾向哪条路径）

**❓ 确认问题：** "倾向测试显示您更适合 [路径]，这个结论是否准确？"

### 步骤 2：市场间隙测试

**关键问题：** 当前市场是否已有同类简化者？空缺是否在能力圈内？

**分析方法：**

```
1. 竞品扫描
   - 列出所有主要竞品
   - 标注其简化路径（价格/命题）
   - 识别市场空白点

2. 细分维度匹配
   - 价格带分布（高/中/低）
   - 用户体验水平（优/中/差）
   - 功能复杂度（简单/中等/复杂）

3. 机会评估
   - 哪条路径竞争较少？
   - 哪个空白点匹配你的能力？
```

**与用户互动：**
```
AI: 请提供以下信息，以便进行市场间隙分析：
    1. 您认为主要竞品是谁？他们的简化路径是什么（价格/命题）？
    2. 您的差异化优势是什么？
    3. 您认为市场空白点在哪里？
    
用户：[回答]

AI: 基于您的回答，市场间隙分析结论：
    - 竞品分布：[分析]
    - 市场空白：[分析]
    - 机会点：[结论]
```

**输出：** 市场间隙分析报告

**❓ 确认问题：** "市场间隙分析显示 [机会点]，您认为这个分析是否准确？"

### 步骤 3：钥匙测试

**关键问题：** 是否握有开启所选路径的核心"钥匙"（独特资源、技术或流程）？

**钥匙类型：**

**价格简化钥匙：**
- 独特的低成本技术或工艺
- 供应链整合能力
- 规模化生产优势
- 渠道成本优势

**命题简化钥匙：**
- 专利性用户体验技术
- 品牌忠诚度
- 生态系统优势
- 数据驱动的个性化能力

**类比方法：**
- 找到成功简化者案例
- 识别其核心钥匙
- 寻找你可复用的模式

**与用户互动：**
```
AI: 请描述您的核心优势/钥匙：
    1. 您有什么独特的资源、技术或流程？
    2. 这些优势如何帮助您实现简化？
    3. 竞争对手难以复制的原因是什么？

用户：[回答]

AI: 基于您的回答，钥匙测试结论：
    - 核心钥匙：[列表]
    - 可持续性：[评估]
    - 可复制性：[评估]
```

**输出：** 钥匙清单及可行性评估

**❓ 确认问题：** "钥匙测试显示您的核心优势是 [列表]，您认为这个分析是否准确？"

### 步骤 4：决策

**关键动作：** 明确简化路径宣言（价格/命题/组合策略）及理由

**简化路径类型：**

| 类型 | 描述 | 适用场景 |
|------|------|---------|
| **价格简化** | 极致低价，功能裁剪 | 成本敏感用户、大众市场 |
| **命题简化** | 极致体验，复杂内核 | 体验敏感用户、高端市场 |
| **组合策略** | 价格+体验双重优势 | 差异化竞争、全面领先 |

**组合策略适用条件：**
- 团队在价格和体验方面都有优势
- 市场需要差异化竞争
- 有足够的资源同时推进两条路径
- 竞争对手只专注一条路径

**与用户互动：**
```
AI: 基于前面的测试结果：
    - 倾向测试：[结论]
    - 市场间隙：[结论]
    - 钥匙测试：[结论]
    
    综合分析，建议选择 [路径]。
    
    请确认您的简化路径选择：
    1. 价格简化（极致低价）
    2. 命题简化（极致体验）
    3. 组合策略（价格+体验）
    
用户：[选择]

AI: 好的，您选择了 [路径]。让我们明确简化宣言。
```

**宣言格式：**
```
我们选择 [价格简化/命题简化/组合策略] 路径，因为：

1. 倾向测试显示：[团队/资源/能力倾向]
2. 市场间隙显示：[市场机会描述]
3. 钥匙测试显示：[核心优势描述]

我们的承诺：[一句话描述简化承诺]
```

**示例：**
```
【价格简化示例】
我们选择价格简化路径，因为：
1. 倾向测试显示：团队有供应链管理经验
2. 市场间隙显示：低端市场空白
3. 钥匙测试显示：自有低成本技术
我们的承诺：让每个人都能买得起的对讲机

【命题简化示例】
我们选择命题简化路径，因为：
1. 倾向测试显示：团队拥有强大的 UX 设计能力
2. 市场间隙显示：竞品体验普遍较差
3. 钥匙测试显示：专利性交互技术
我们的承诺：让复杂的技术 invisible

【组合策略示例】
我们选择组合策略，因为：
1. 倾向测试显示：团队在成本和体验方面都有优势
2. 市场间隙显示：需要差异化竞争
3. 钥匙测试显示：成本技术+UX能力都具备
我们的承诺：极致低价+极致体验
```

**❓ 确认问题：** "简化宣言为：[宣言内容]，您是否确认这个方向？"

### 步骤 5：行动

**根据决策立即启动：**

**如果选择价格简化：**
- ✅ 立即启动功能裁剪
- ✅ 开始低成本供应链设计
- ✅ 规划规模化生产路径

**如果选择命题简化：**
- ✅ 同步建设用户体验闭环
- ✅ 建立快速迭代机制
- ✅ 规划品牌升级路径

### 风险提示

**需要重新评估的情况：**
- 三项测试结果不一致
- 市场已被充分占领
- 缺乏核心钥匙

**应对策略：**
- 重新评估或考虑收购/合作防御
- 寻找差异化细分市场
- 通过合作获取关键资源

### 核心输出

**选择层输出：**
- ✅ 清晰的简化路径宣言
- ✅ 落地资源清单
- ✅ 风险评估报告

## 第二阶段：设计层

### 目标

通过重构产品/服务，实现"把复杂留给自己，把简单留给用户"

### 路径 A：价格简化设计

**核心逻辑：** 以"极致低价"为目标，通过砍功能、降成本实现规模化优势

#### 关键步骤 1：功能裁剪

**从第一原则出发：**
- 识别用户最核心的 1-2 个需求
- 剔除非核心功能
- 创造"通用产品"替代多品类

**裁剪清单模板：**
```
| 功能 | 核心价值 | 使用频率 | 复杂度 | 决策 |
|------|---------|---------|--------|------|
| 功能 A | 高 | 高 | 低 | 保留 |
| 功能 B | 中 | 低 | 高 | 删除 |
| 功能 C | 低 | 低 | 中 | 删除 |
```

**案例参考：**
- 福特 T 型车：仅保留基础驾驶功能
- 宜家平板包装：标准化替代定制化

#### 关键步骤 2：系统重构

**优化供应链：**
- 简化生产流程
- 降低制造成本
- 提高标准化程度

**规模化生产：**
- 通过高销量摊薄单位成本
- 建立成本领先优势

**案例参考：**
- 麦当劳标准化食材配送
- 亚马逊低价电子书策略

#### 输出

**价格简化设计输出：**
- ✅ 功能裁剪清单
- ✅ 低成本供应链蓝图
- ✅ 规模化生产路线图

### 路径 B：命题简化设计

**核心逻辑：** 以"极致易用"为核心，通过复杂内核提升用户体验

#### 关键步骤 1：用户视角重构

**优先解决"易用性"痛点：**
- 绘制用户旅程地图
- 识别所有摩擦点
- 用复杂过程换取消费端简便

**设计原则：**
- 先易用后美观
- 零学习成本
- 无负担使用

**案例参考：**
- 苹果鼠标：消除用户操作门槛
- 宜家自提家具：降低安装难度

#### 关键步骤 2：体验升级

**提升实用性与美观性：**
- 功能与美学平衡
- 情感化设计
- 品牌一致性

**闭环设计：**
- 从使用到维护全程无负担
- 建立反馈迭代机制

**案例参考：**
- 无印良品：简约设计兼顾功能与审美
- 戴森吸尘器：可拆卸滤网设计

#### 输出

**命题简化设计输出：**
- ✅ 用户旅程地图
- ✅ 体验迭代计划
- ✅ 闭环设计方案

### 设计层验证

**验证清单：**
- [ ] 设计与选择层路径一致
- [ ] 设计与执行层强关联
- [ ] "复杂留给自己"原则落地
- [ ] 用户价值清晰可感知

**提示：** 设计层必须与后续执行层强关联，避免"设计与系统脱节"

### 智能功能设计（如适用）

**如果产品包含智能功能（如 AI 助手），使用以下框架：**

#### 智能功能定位

| 维度 | 问题 | 决策 |
|------|------|------|
| **角色定位** | 智能功能是"核心卖点"还是"增值功能"？ | 核心→重点投入，增值→控制成本 |
| **技术架构** | 本地处理还是云端处理？ | 本地→增加成本，云端→依赖网络 |
| **用户价值** | 解决什么实际问题？ | 必须回答，否则删除 |
| **成本影响** | 增加多少 BOM 成本？ | 必须可控 |

#### 智能功能设计原则

**端云协同设计：**
- 简单任务本地处理（降低成本）
- 复杂任务云端处理（保证体验）
- 明确分工，平衡成本与体验

**与极简法则的结合：**
- 智能功能也要"简化"：只做有价值的事
- 用户无感知：让复杂技术隐藏在后端
- 学习用户习惯：减少重复操作

#### 输出

**智能功能设计输出：**
- ✅ 智能功能定位报告
- ✅ 端云协同架构图
- ✅ 功能清单（保留/删除/简化）
- ✅ 成本影响评估

### 组合策略设计（如适用）

**如果选择组合策略，同时进行价格简化设计和命题简化设计：**

#### 设计重点

| 维度 | 价格简化 | 命题简化 | 组合策略 |
|------|---------|---------|---------|
| **功能** | 只保留核心功能 | 保留核心+体验功能 | 核心功能+极致体验 |
| **成本** | 极致成本控制 | 成本服从体验 | 成本控制但不牺牲体验 |
| **体验** | 基础可用即可 | 极致体验 | 基础体验+亮点体验 |
| **用户** | 价格敏感用户 | 体验敏感用户 | 全面覆盖 |

#### 输出

**组合策略设计输出：**
- ✅ 价格简化设计
- ✅ 命题简化设计
- ✅ 优先级清单（哪些功能必须保留，哪些可以裁剪）

### 核心输出

**设计层总输出：**
- ✅ 一份符合企业能力的简化路径（价格/命题/组合策略）
- ✅ 一套可落地的设计原则（功能裁剪清单/用户体验地图）
- ✅ 设计与执行层的衔接方案
- ✅ 智能功能设计（如适用）
- ✅ 组合策略设计（如适用）

### 文档输出

**将规划结果写入：** `docs/plans/YYYY-MM-DD-<topic>-plan.md`

**文档包含章节：**
1. 执行摘要（简化路径宣言）
2. 选择层分析（倾向测试、市场间隙测试、钥匙测试）
3. 设计层方案（功能裁剪/用户体验设计）
4. 智能功能设计（如适用）
5. 组合策略设计（如适用）
6. 设计与执行衔接方案
7. 风险评估与应对
8. 下一步行动

### 自动流程：询问下一步

**完成规划后，必须执行以下询问流程：**

```
输出所有规划结果（路径宣言 + 设计方案）
    ↓
询问用户：
    ↓
"规划已完成！我们确定了 [简化路径]，并完成了产品设计方案。

接下来是否要执行 ESP-Act（执行与防御技能），建立专属系统、加速扩张并建立防御机制？

回复'是'或'开始'将继续执行 Act 技能。"
    ↓
等待用户确认
    ↓
如果用户确认 → 自动调用 ESP-Act 技能
如果用户拒绝 → 暂停，等待进一步指示
```

**询问话术示例：**
```
✅ 规划已完成！

本次成果：
- 简化路径：命题简化
- 核心优势：专利性交互技术 + 强大 UX 团队
- 设计输出：用户体验地图（5 个关键优化点）

接下来是否要执行 ESP-Act（执行与防御技能），建立专属系统、加速扩张并建立防御机制？

回复"是"或"开始"将继续执行 Act 技能。
```

**注意事项：**
- 必须等待用户明确确认后才能继续
- 如果用户拒绝，记录原因并暂停
- 如果用户要求修改，先满足修改需求再询问

## 使用方法

### 完整流程

1. **先跑选择层**
   - 完成倾向测试、市场间隙测试、钥匙测试
   - 输出简化路径宣言

2. **并行启动设计层**
   - 根据选择层决策，启动对应路径的设计
   - 输出功能裁剪清单或用户体验地图

3. **确保强关联**
   - 设计与选择层一致
   - 设计可落地到执行层

4. **准备交接**
   - 整理完整设计文档
   - 为执行层提供清晰指引

### 常见错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **测试不充分** | 跳过某项测试直接决策 | 严格执行三项测试，缺一不可 |
| **路径模糊** | 宣言同时包含价格和体验 | 强制选择一条路径（价格/命题/组合） |
| **设计脱节** | 设计与执行层无法衔接 | 设计阶段邀请执行团队参与 |
| **缺乏验证** | 没有验证设计是否符合路径 | 使用验证清单逐项检查 |
| **忽视智能功能** | 产品有智能功能但未专门评估 | 增加智能功能设计章节 |

## 完整流程示例

### 场景：智能对讲机产品规划

**步骤 1：倾向测试**
```
AI: 【倾向测试】
    请评估您的团队/企业在哪个方面更强：
    - 团队专长：供应链管理 vs 用户体验设计
```
